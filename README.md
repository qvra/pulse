# pulse

Heartbeat surface for `qvra`.

`pulse` is the visible rhythm of the field.
It exists so people can return and see that motion is real, current, and directional.

## What belongs here

Artifacts in `pulse` should provide one or more of:
- release summaries
- motion logs
- field heartbeat updates
- notable shifts
- live route changes
- current highlights worth revisiting

## What does not belong here

`pulse` is not:
- passive archive prose
- duplicated planning text
- empty cadence theater
- historical clutter competing with live movement

## Anchor standard

Anything published here should make one of these clearer:
- what moved
- what improved
- what appeared
- what changed direction
- what deserves attention now

## Current role in qvra

`pulse` is one of the gravitational anchors.
It sustains recurrence, return traffic, and public proof that the field is alive.

Related routes:
- [`run`](https://github.com/qvra/run) for immediate utility
- [`bench`](https://github.com/qvra/bench) for measured pressure
- [`lab`](https://github.com/qvra/lab) for experiments and forward edges
- [`show`](https://github.com/qvra/show) for visible demonstrations
- [`watch`](https://github.com/qvra/watch) for more curated observation later

## Rhythm rule

If `pulse` stops reflecting real movement, the field starts reading as inventory.
